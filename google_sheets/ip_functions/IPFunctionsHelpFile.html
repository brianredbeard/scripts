Overview

IPFunctions is a set of Google Apps Spreadsheet Scripts to automate common IP address management functions. The functions are broken into two basic types of functionality. One is user defined spreadsheet functions that allows the use of IP-specific functions in spreadsheet formulas. The second is a new menu that allows for other types of menu-driven tasks from within the spreadsheet.

The user-defined functions for IP address management can be used in formulas like any other functions such as SUM(). The IP-specific functions allow you to perform similar actions on IP addresses, such as IPADD() allows you to add a decimal number to an IP address. There are also unique IP-specific functions that perform actions such as generating the IPMASK(), or returning the IPBROADCAST() address. All of these functions will be detailed in this doc.

The menu items perform a few IP-specific functions, like IP FILL which fills in a range of IP address in IP order - meaning for example, that it will not increment an octet past .255, and will know how to wrap octets from .0.255 to .1.0 and beyond. Also some other useful spreadsheet management tools, like converting to lowercase and removing formulas.

IP Functions User Defined Functions Specifications

All IP Functions work with standard CIDR IPV4 address notation. This means all address will be specified as text strings with a subnet mask. Some functions may accept addresses without the mask if the mask is not necessary for the function.

IP Management Functions:

IPADD(ip, int)

Takes an IP address and adds an integer. This will work across subnet boundaries. If no integer is provided, it will increment the IP by 1.

IPADDR(ip)

Takes an IP/mask in CIDR notation and returns just the IP address.

IPBROADCAST(ip)

Takes an ip/mask in CIDR notation and returns the broadcast address.

IPFILL(ip)

Takes an ip/mask in CIDR notation and returns an array in the same column of all hosts enumerated in the subnet.

IPHOSTS(ip)

Takes an ip/mask in CIDR notation and returns the number of available hosts.

IPISIN(ip1, ip2)

Returns true if ip1 is contained within ip2. Ip2 must be an ip with subnet mask in CIDR notation. Ip1 can be either a single ip address or a subnet in CIDR notation.

IPMASK(ip)

Takes an ip/mask in CIDR notation and returns the mask value in dotted decimal, such as 255.255.255.0.

IPMASKLEN(ip)

Takes an ip/mask in CIDR notation and returns only the mask length.

IPNETWORK(ip)

Takes an ip/mask in CIDR notation and returns the network address.

IPNEXTNET()

Takes an ip/mask in CIDR notation and returns the next matching network with the same subnet mask.

IPOCTET(ip, octet)

Takes an IP address with or without CIDR notation and returns the specific octet specified by ‘octet’. If no octet is specified, it will return an array with all 4 octets.

IP Conversion Functions

IPBIN2DD(ipbin)

Converts 32-bit binary (0’s and 1’s string) to an IP address in dotted decimal notation.

IPDD2BIN(ip)

Converts IP in dotted decimal notation to binary string.

IPDD2DEC(ip)

Converts IP in dotted decimal notation to decimal number. Useful for sorting and mathematical operations.

IPDD2HEX(ip)

Converts IP in dotted decimal notation to HEX string.

IPDEC2DD(ipdec)

Converts decimal number to IP address in dotted decimal notation.

IP Functions Menu

Not all functions in the menu are fully implemented.

IPFILL

If a single cell containing an IP address in CIDR notation is selected, then IP FILL will fill in the cells below with all the IP address in the subnet. If a range is selected it will incrementally fill the range starting from the top IP address. If a ROW is selected instead of a COLUMN, it will perform the same operation across the columns.

SUBNET FILL

Similar to IP FILL above except it will fill the subnets incrementally but only works if a range is selected. For example if 10.10.10.0/26 is selected, it will fill the sleected range of cells with 10.10.10.64/26, 10.10.10.128/26 etc... This will work fur subnets and supernets it does not care.

IP SORT

Will take a selected range of ip addresses and sort them by IP address. This is very useful such as:

TO LOWER

Converts the selected range to all lower case.

REMOVE FORMULAS

Removes all formulas from the selected range. Especially useful with google sheets where performance can be seriously impacted by too many formulas. Also useful when copying and pasting data to another spreadsheet to get rid of formulas.
